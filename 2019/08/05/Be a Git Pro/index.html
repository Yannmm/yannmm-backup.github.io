Merge & Rebase

Merge

  • 如果两个 commit 处于同一条历史上，没有分支，则二者将会以 fast-forward 的形式 merge，不产生新的 commit
  • 两个 commit 位于不同分支，则这是一个三方merge，tip1 + tip2 + common ancestor，产生新 commit，有两个parent
  • 发生冲突时，冲突的部分将会添加 marker，以便我们手动解决。解决后，应该通过 git add 添加到 staging area，表示冲突已解决（冲突其实就是 git 对文件做了修改，要求我们再次修改为正确的，文件的状态从 unmodified > modified > staged）
  • 将 base branch rebase 至 target branch
  • 不产生新的 commit
  • 结果就是所有 commit 都是线性的，没有分叉
  • rebase 后，target branch 和 base branch merge时，只需 fast-forward 即可
  • merge过后，base branch 可以删除
  • 绝对注意，只 rebase 私有分支，千万不要对公共分支操作

Rebase 的步骤

  1. 找到 common ancestor
  2. 从 common ancestor 开始diff
  3. 将当前分支指向目标分支 tip
  4. 然后依次将 diff apply

Rebase vs. Merge

rebase 本地分支，merge 公共分支
